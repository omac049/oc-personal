Site Health (WordPress)

From your Site Health report. Addressing these items will improve security, updates, and performance.

2 critical Prioritize first
8 recommended Beneficial to address

Critical issues

  • WPForms license expired — Renew at wpforms.com/account or remove/replace WPForms. Without a valid license you lose updates, security fixes, and support.
  • Background updates not working — Often tied to REST API / loopback returning 403. Fix REST API and loopback (security plugin or host) so core and plugins can auto-update.

Recommended improvements

  • WordPress update available (6.9.1) — Install via Dashboard → Updates after a backup.
  • Remove inactive plugins & update plugins — 14 plugins need updates; 14 inactive. Update all active; delete inactive ones you don’t need (or document for approval).
  • Default theme — Install one default theme (e.g. Twenty Twenty-Four) and leave inactive as a fallback if the active theme fails.
  • Scheduled event failed (action_scheduler_run_queue) — Usually fixed once loopback/REST API work; then re-check Site Health.
  • REST API 403 Forbidden — Allow /wp-json in Wordfence (or similar) and host firewall; block editor and cron depend on it.
  • Persistent object cache — Host supports Redis/Memcached; enabling one (via host + plugin) will speed up the site.
  • Loopback request failed (403) — Same fix as REST API: allow requests to your own site URL (security plugin / host).
  • Page cache not detected — May be a side effect of loopback failure; after fixing loopback, confirm Hostinger or a cache plugin is serving cache headers.

PHP version (Hostinger)

Hostinger reports that your current PHP version (8.3) is not compatible with your current WordPress version. This can cause errors and should be addressed during stabilization.

  • Action — In Hostinger hPanel, set PHP to a version compatible with your WordPress release (e.g. PHP 8.1 or 8.2; check WordPress PHP requirements). Typically: Advanced → PHP Configuration (or similar), select the PHP version for this site. We’ll include this in Phase 1 scope.
